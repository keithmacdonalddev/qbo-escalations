### 1. MEDIUM (static, R02) — Fix dangling `scripts/scope.sh` path in cto-review

**Evidence:**
- `cto-review/SKILL.md:45` — `bash ${CLAUDE_SKILL_DIR}/scripts/scope.sh <plan-path-or-empty>`
- `cto-review/SKILL.md:62` — "edit the `classify()` function in `scripts/scope.sh`"
- Actual location: `cto-review/scope.sh` (script is at the skill root, not in `scripts/`).
- `cto-review/scripts/` directory does not exist.

**Why it matters:** If/when the harness expands `${CLAUDE_SKILL_DIR}` to the skill folder and runs the bash command, it will fail with "No such file or directory." The script exists; the path in the SKILL.md is wrong.

**Two options:**

- **Option A (preferred):** Create `cto-review/scripts/` and move `scope.sh` into it. Matches the standard skill layout pattern referenced by SC03 and SC04 rules.
- **Option B:** Update SKILL.md lines 45 and 62 to reference `${CLAUDE_SKILL_DIR}/scope.sh` (no `scripts/` prefix).

Option A is cleaner because it leaves room for future helper scripts and matches the layout `examples/`, `references/`, `scripts/`. Effort: trivial — one `mkdir` and one `git mv` and bump the cto-review version header.

**Proposed edit:**
- Run `mkdir -p .claude/skills/cto-review/scripts && git mv .claude/skills/cto-review/scope.sh .claude/skills/cto-review/scripts/scope.sh`.
- Bump cto-review SKILL.md version header to `0.5` and update the date.

This is a known issue pattern. Here's a draft response for the phone agent:

Resolution Steps:

  1. Navigate to Payroll > Employees > Paycheck List
  2. Filter by the affected pay period to confirm the duplicate entry
  3. If only one paycheck shows on Intuit's side, the duplicate is a display sync artifact
  4. Have the customer go to Settings > Payroll Settings > Sync and run a manual sync
  5. If the duplicate persists after sync, void the ghost entry (it will not affect the actual payroll submission)

Important: Before voiding, verify the payroll tax filings are correct on the Intuit Payroll side. Voiding a real paycheck versus a display ghost has very different consequences.

Issue Summary: Customer at Maple Ridge Construction reports duplicate payroll entries appearing in QBO after running payroll. Investigation reveals the customer initiated payroll processing twice, but only one submission was recorded on the Intuit payroll service side.

Root Cause: This is a display synchronization artifact. The QBO client-side cached the first payroll run and created a phantom duplicate when the second run was initiated. The underlying payroll data and tax filings are correct.

Recommended Resolution:

  1. Verify on Intuit Payroll side that only one paycheck exists per employee
  2. Navigate to Paycheck List and identify the ghost entry
  3. Run manual sync from Settings > Payroll Settings
  4. If ghost persists, void the duplicate (no tax impact)

Risk Level: Medium -- voiding the wrong entry could affect tax filings. Advise agent to confirm paycheck IDs before voiding.

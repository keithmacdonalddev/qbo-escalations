Design Philosophy

Filing Cabinet → Command Center

The old plan built a tree nav + split-pane editor. That's what authors need. But the primary users aren't authors — they're escalation specialists on live calls with 3 more queued, hearing "my bank won't connect" while a frustrated CPA waits. They need answers in 3 seconds, not a file browser.

The new plan puts Lookup Mode first — a rapid-response console with Answer Cards. Edit Mode becomes secondary. A persistent Call Companion bar follows you everywhere. A Chat Sidebar shows what KB content Claude is reading in real-time. The tool works at the speed of conversation.

Before (Filing Cabinet)

"I know the answer is somewhere in the playbook. Let me navigate to bank-feeds... scroll down... find the right section... okay, here it is. Let me copy this and go back to chat."

30-60 seconds. Tab switching. Scrolling. Stress.

After (Command Center)

Type "bank won't connect" in the bottom bar while Claude is still streaming. Answer card pops up. Click "Copy Steps." Paste. Done.

3 seconds. No tab switching. Never left Chat.

Phase 1 — Command Center Foundation

1 Lookup Mode + Answer Cards + Call Companion + Full API

Build the rapid-response console, persistent bottom bar, and backend API for all 22 files. The phase that transforms the tool.

Lookup Mode

Default playbook view. Single dominant search input: "What's the issue?" Results appear as Answer Cards — not file links. Each card: heading, category badge, numbered steps (max 6), "Copy Steps" button, "Agent Script" button, cross-refs. One screen per card. Sub-200ms, no AI call.

Call Companion Bar

Persistent bottom bar on every screen — Chat, Dashboard, everywhere. Quick lookup input with flyout popup. Clipboard history (last 3 copies). Call timer. Active escalation badge. The specialist never leaves their current view to search the KB.

Edit Mode

Refactored PlaybookEditor as secondary mode. Tree nav with 4 sections (Categories, Templates, Reference, System). All 22 files viewable and editable. Metadata bar with word count, token estimate, last modified. Toggled from Lookup Mode.

Phase 2 — Canada-First Content

2 Knowledge Base Expansion: 2.6K → 15K Lines

Rewrite all categories Canada-first. Create 7 new files. Directly improves Answer Card quality.

Every Line Must Be Canada-First

CRA not IRS. GST/HST/PST/QST not US sales tax. CPP/EI not FICA/FUTA. T4/T4A not W-2. ROE not termination letter. Quebec gets special treatment: RRQ, QPIP, CNESST, RL-1, QST. Know US features only to say "that's US-only."

Phase 3 — Context Integration + Editor

3 Chat KB Sidebar + Diagnostic Wizards + Editor Polish

The KB actively helps during conversations. Interactive diagnosis flows replace reading markdown trees.

Chat KB Sidebar

Collapsible right panel in Chat. Auto-surfaces relevant Answer Cards based on the conversation. Specialist sees what KB content Claude is reading — can verify AI responses against source. Updates per message. No manual search.

Diagnostic Wizards

Click-through guided flows: "Bank issue" → "Which bank?" → [TD] → "Disconnected or wrong txns?" → resolution card. Defined in playbook/diagnostics/*.yaml. Wizard updates when playbook content is edited.

Editor Polish

Split-pane markdown editor. Ctrl+K command palette. Ctrl+S save. Auto-save drafts to localStorage. Unsaved changes warning on sidebar nav. Stats dashboard with token counts.

Phase 4 — Smart Search

4 Stemming + BM25 + Canadian Synonyms

Directly improves Lookup Mode, Chat sidebar, and Call Companion quality. Backend-only changes.

Current Search

  • Exact word match only
  • Hand-tuned magic numbers
  • "payment" misses "payments"
  • "GST" misses "goods and services tax"

After Phase 4

  • Porter stemmer — inflection-aware
  • BM25 scoring (k1=1.5, b=0.75)
  • Synonym map — Canadian terms expanded
  • Answer Cards get noticeably better

Phase 6 — Self-Improving KB

6 Learning Loop + Gap Detection + Confidence Scoring

The KB grows from every resolved escalation. Gaps surface automatically. Knowledge compounds.

Learning Loop

After resolving an escalation: "This fix isn't in the playbook." One-click "Propose Addition" opens editor pre-filled at the right section. Knowledge flows from field to KB.

Gap Detection

AI analyzes conversations where Claude fell back to generic advice. Generates gap cards: topic, frequency, suggested content. "14 resolutions this month aren't in the playbook."

Confidence Scoring

Track retrieval frequency × resolution correlation per article. Low-confidence flagged for expansion. High-confidence boosted in search. KB quality becomes measurable.

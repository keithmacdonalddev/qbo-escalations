Dev Mode Mini Widget

background process monitoring — design exploration

1 — The Problem

Navigating away kills in-flight Dev Mode processes

When you switch from the Dev Mode tab to any other tab, React unmounts the DevMode component. This triggers a cleanup chain that terminates the running subprocess and discards the partial response.

Navigate away
→
Component unmounts
→
SSE aborted
→
Subprocess killed
→
Response lost
Root cause

App.jsx uses a switch statement to render views — only the active tab's component is mounted. When DevMode unmounts, useDevChat's cleanup effect calls abortRef.current(), closing the SSE connection. The server's req.on('close') handler then sends SIGTERM to the Claude/Codex child process.

The solution

Keep DevMode mounted (hidden via CSS) and add a mini widget that surfaces streaming progress while you’re on other tabs. The SSE connection stays alive, the subprocess runs to completion, and you get a live preview of what’s happening.

3 — Widget States

The widget transitions through five states based on the dev process lifecycle. It only appears when there’s something to show.

Hidden

No active dev process. Widget not rendered.

Streaming

Process running. Spinner + live text preview of the last few lines.

Tool Use

Shows tool event badges: Read Edit Bash

Complete

Green checkmark. Auto-fades after 5s or dismiss on click.

Error

Red indicator with error message. Click to dismiss or open full view.

4 — Implementation Architecture

The hook moves up to App.jsx. Both DevMode and DevMiniWidget consume the same state. The widget only appears when there’s an active stream and the current tab is not Dev Mode.

App.jsx — Hook Owner
Calls useDevChat() at the top level. Owns all streaming state, SSE connection, and abort ref. Passes the full hook return value as props to both consumers.
props passed down
DevMode
Full dev mode view. Always mounted, hidden with display: none when route.view !== 'dev'. Receives hook state as props.
DevMiniWidget
Compact floating widget. Renders when isStreaming && route.view !== 'dev'. Shows streaming text preview, tool events, provider badge. Click navigates to #/dev.
visibility logic
Render Logic
App.jsx changes:
  • useDevChat() moved from DevMode into App.jsx
  • DevMode rendered always (not inside switch), hidden via CSS when inactive
  • DevMiniWidget rendered at root level, conditionally visible
  • All other views stay in the switch as-is — no changes
Key implementation details
  • No backend changes needed — the SSE connection simply stays alive because the component never unmounts
  • DevMode receives hook state as props instead of calling useDevChat() internally
  • DevMiniWidget is a new, lightweight component (~100 lines) with its own collapsed/expanded toggle
  • The widget’s “Open Dev Mode” button sets window.location.hash = '#/dev'
  • After streaming completes, the widget shows a success state for 5 seconds then auto-hides

6 — Recommendation

Option A: Floating Bottom-Right Widget

The floating widget is the strongest choice for background process monitoring. It provides the right balance of visibility without intrusion, and follows a well-understood interaction pattern that users already know from chat widgets.

✓ Zero layout disruption — absolutely positioned, doesn’t affect sidebar or content flow
✓ Always accessible — visible from any tab regardless of sidebar state
✓ Familiar pattern — Intercom, Crisp, and chat bubbles have trained this interaction
✓ Progressive disclosure — collapsed pill for at-a-glance, expanded for detail
✓ Easy to dismiss — auto-fades after completion, or close manually
✓ Simplest implementation — ~100 LOC new component, no existing component changes beyond App.jsx
Implementation summary

Three changes needed:

  • App.jsx — Lift useDevChat() to App level. Render DevMode always (hidden when inactive). Add <DevMiniWidget /> at root.
  • DevMode.jsx — Accept hook state as props instead of calling useDevChat(). No other changes.
  • DevMiniWidget.jsx — New component. Floating pill with collapsed/expanded states. Reads isStreaming, streamingText, toolEvents, provider from props.

Unanimous Verdict

The Playbook is a Skeleton Wearing the Label "Knowledge Base"

All three reviewers independently reached the same conclusion: the playbook has good structural bones (consistent category format, working inference pipeline, functional CRUD) but the content is a surface-level cheat sheet — the kind you'd give a new hire on day one, not what a senior escalation specialist relies on. The editor is a textarea wrapper that can only reach 10 of 22 files. The search exists in backend code but is invisible to users. There are zero tests. The system is architecturally sound but content-starved, management-crippled, and untested.

Backend Architect — Critical Findings

Architecture Strengths

  • Inference pipeline is solid — Loader → context builder → Claude subprocess → copilot is well-designed with token budgeting and 3 knowledge modes
  • Semantic chunking — 1400-char paragraph-aware chunks with unique IDs
  • File watching — Changes auto-reload without server restart
  • Context builder flexibility — full-playbook, retrieval-only, core+retrieval modes with multi-stage budget enforcement

Architecture Failures

  • Won't scale — Entire playbook loaded into memory. At 25K lines (~150KB), system prompt becomes 37K+ tokens per request. No chunked retrieval strategy for large KB
  • Lexical search only — No stemming, no synonyms, no fuzzy matching. "payment" won't match "payments". No semantic understanding
  • No content versioning — Files on disk, no history, no rollback, no audit trail of who changed what
  • 13 of 22 files unreachable via API — Templates, system-prompt, and reference docs have no endpoints

Innovative Features — What Nobody Thought Of Yet

Cross-pollinated from all three reviewers — features that would make this KB system unlike anything else in the support tooling space.

AI Content Gap Detector

Analyze incoming escalation conversations against playbook content. When Claude struggles or falls back to generic advice, flag the topic as a knowledge gap. Auto-generate "gap cards" with: topic, frequency, example conversations, suggested content outline. The playbook grows organically from real support needs.

Resolution Confidence Scoring

For each playbook article, track: how often it's retrieved, how often conversations using it reach resolution, customer satisfaction correlation. Display a confidence score (0-100%) on each article. Low-confidence articles get flagged for expert review and expansion. High-confidence articles get promoted in search ranking.

"How Will Claude See This?" Mode

A preview mode showing exactly how the AI will process the content: token count, chunk boundaries, retrieval ranking simulation for sample queries, system prompt position. Lets authors optimize content for AI consumption — not just human readability. Shows which paragraphs are most likely to be retrieved for different question types.

Seasonal Intelligence Engine

Auto-surface relevant KB sections based on calendar: W-2/1099 in January, quarterly tax in April/July/October, year-end close in December. Pre-brief specialists before predictable surges. Track historical escalation volume by topic by month to predict next surge.

Escalation-to-KB Pipeline

When an escalation is resolved with a novel fix, offer a one-click "Add to Playbook" button. Auto-extracts: problem description, diagnosis steps, resolution, and generates a draft KB article in the right category. Closes the loop between field experience and documented knowledge.

Content Freshness Radar

Track QBO product updates (release notes, changelogs) and cross-reference against playbook content. When a feature changes, auto-flag affected articles as potentially stale. Example: "QBO changed bank feed reconnection flow in Feb 2026 — bank-feeds.md may be outdated." Prevents stale advice.

Quick-Answer Cards

Distill each category into top-10 instant answer cards — one-screen answers to the most common questions. During live calls, agents can pull up a card in <5 seconds, read the answer, and resolve without scrolling through full articles. Cards link to full articles for complex cases.

Knowledge Graph Visualization

Visualize relationships between KB articles as an interactive graph. See how payroll connects to tax, how bank feeds connects to reconciliation, how permissions affects everything. Helps authors identify missing cross-references and helps specialists understand systemic issues.

Difficulty-Aware Content Layers

Tag content as Beginner / Intermediate / Expert. In the UI, show appropriate depth based on the specialist's experience level. New hires see step-by-step guides; seniors see decision trees and edge cases. AI gets everything, but surfaces the right depth for context.

Comparative Diagnostics

For complex issues, provide side-by-side comparisons: "If customer sees X, it could be Problem A, B, or C. Here's how to distinguish." Decision tables with yes/no diagnostic questions that narrow to the exact issue. Turn the KB into an interactive diagnostic tool, not just a reference.

Unified Build Roadmap

Synthesized from all three reviews into a phased plan. Each phase builds on the previous and can be shipped independently.

1 Foundation — Make All Content Manageable

Unblock content work by making all 22 files editable through the UI. No content expansion possible until this is done.

  • 1 Backend: Template CRUD routes — GET/PUT for templates/*.md, mirroring category pattern
  • 2 Backend: Reference doc routes — GET/PUT for top-level .md files (system-prompt, triage, error-messages, decision-tree, qbo-urls, translations)
  • 3 Backend: Search endpoint — Expose searchPlaybookChunks() as GET /api/playbook/search?q=
  • 4 Backend: Status endpoint — GET /api/playbook/status with file count, total tokens, chunk count, last rebuild
  • 5 Frontend: Redesign PlaybookEditor — Replace 3-tab layout with tree-based navigation covering Categories, Templates, Reference Docs, System Config
  • 6 Frontend: Wire all new endpoints — API client + UI for templates, reference docs, search
  • 7 Backend: Fix error handling — try/catch on all file ops, specific error messages, watcher debouncing

2 Content Expansion — Grade School → University Level

Massive content depth expansion across all 9 categories + new cross-cutting knowledge areas.

  • 1 Expand each category to 500-800+ lines with deep troubleshooting trees, exact QBO UI paths, accounting principles, version-specific behaviors
  • 2 Add accounting fundamentals layer — GAAP basics, double-entry theory, accrual vs cash, chart of accounts design, journal entry principles
  • 3 Add cross-cutting knowledge areas — Data migration, multi-entity, integrations, mobile app, QBO Advanced features
  • 4 Expand templates to dynamic templates — Conditional sections, tone variants, multi-audience versions
  • 5 Add diagnostic decision tables — Interactive "if X then Y" diagnostic flows for each common issue cluster
  • 6 Populate Known QBO Bugs sections — Research and document current known issues with workarounds
  • 7 Add seasonal awareness content — Monthly/quarterly calendar of predictable issue surges with pre-briefing content

3 Editor Excellence — Make Authoring a Pleasure

Transform the textarea into a world-class KB editor that authors want to use.

  • 1 Markdown split-pane editor — Source on left, rendered preview on right, with formatting toolbar
  • 2 Command palette (Ctrl+K) — Fuzzy search overlay for instant content lookup during live calls
  • 3 KB stats dashboard — Token counts per section, total prompt size, depth scores, freshness indicators, coverage gaps
  • 4 Auto-save + undo/redo — localStorage drafts, Ctrl+Z/Y history, "unsaved changes" protection on all navigation
  • 5 Keyboard shortcuts — Ctrl+S save, Ctrl+E edit, Ctrl+K search, Esc cancel, keyboard nav between sections
  • 6 Cross-reference insertion — When editing, autocomplete for linking to other articles (like wiki links)
  • 7 "How Will AI See This?" preview — Token count, chunk boundaries, retrieval ranking preview for sample queries

4 Smart Search — From Keyword Matching to Knowledge Retrieval

Upgrade search from basic token matching to intelligent knowledge retrieval.

  • 1 Porter stemmer integration — "reconciling" matches "reconciliation", "payments" matches "payment"
  • 2 BM25 scoring — Replace hand-tuned magic numbers with proven information retrieval scoring
  • 3 Synonym expansion — "bank feed" = "bank connection" = "financial institution link"
  • 4 Hierarchical chunking — Preserve section → subsection → paragraph context in search results
  • 5 Usage-weighted ranking — Boost articles that led to successful resolutions, demote unused content
  • 6 Semantic search (stretch) — Embedding-based similarity for concept matching when exact words don't appear

5 Content Expansion II — University → Expert Level

Push content depth to doctorate/expert level with institutional knowledge and advanced patterns.

  • 1 Expand each category to 1000-1500+ lines — Full diagnostic decision trees, every error code mapped, version history of UI changes
  • 2 Regulatory/compliance layer — State-specific tax rules, employment law basics for payroll, data privacy requirements
  • 3 Institutional memory — Historical patterns, "what broke in Q4 2025", customer psychology profiles, de-escalation scripts
  • 4 Root cause taxonomy — Classify every issue into: user error, product bug, configuration issue, data corruption, environmental, integration failure
  • 5 Difficulty-layered content — Tag content as Beginner/Intermediate/Expert, show appropriate depth based on agent experience
  • 6 Comparative diagnostic tables — "If X, could be A, B, or C — here's how to tell" for every common symptom

6 Intelligence Layer — Self-Improving Knowledge Base

The playbook becomes a living, learning system that improves itself from real usage.

  • 1 AI Content Gap Detector — Analyze failed/weak AI responses to identify missing KB content
  • 2 Escalation-to-KB Pipeline — One-click "Add to Playbook" from resolved escalations
  • 3 Resolution Confidence Scoring — Track which articles correlate with successful resolutions
  • 4 Content Freshness Radar — Flag potentially stale content when QBO product updates occur
  • 5 Seasonal Intelligence — Auto-surface relevant content based on calendar patterns
  • 6 Knowledge Graph — Visualize and navigate article relationships, identify missing connections
